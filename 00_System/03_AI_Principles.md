# 🤖 工程严谨与 AI 工作原则 
## 1. 代码绝对正确性 (Code Accuracy) 
- 示例代码必须是可运行的、符合当前行业最佳实践的。严禁在代码示例中使用已被废弃（Deprecated）的 API 或语法。 
- 严禁为了凑字数而捏造不存在的库函数（Zero Code Hallucination）。

# 2. 上下文自包含 (Self-Contained Completeness) 
- 渐进学习要求学习者在复习单张卡片时不需要到处翻找资料。在提取素材时，必须保证“概念-示例-问题”的逻辑闭环，**宁可卡片稍微长一点，也绝不能让核心实现细节丢失。** 

## 3. 边界条件提示 (Edge Cases) 
- 在 CS 领域，“坑”往往比常规用法更重要。如果有明显的内存泄漏风险、并发竞争条件（Race Condition）或性能瓶颈，必须在“检验与思考”或示例注释中显式标注。
